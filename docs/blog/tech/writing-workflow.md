# 我想要的写作工作流

我希望写博客这件事足够轻，不要有太高的启动成本。

理想中的流程大概是这样：

1. 想到主题时先记下来
2. 用 Markdown 快速起草
3. 后续再慢慢打磨结构和表达
4. 发布之后继续迭代

## 为什么选 Markdown

- 写得快
- 可读性高
- 迁移成本低
- 适合长期维护

## 接下来的优化方向

- 补充文章封面
- 增加评论系统
- 接入自定义域名
- 完善 SEO 与站点信息
